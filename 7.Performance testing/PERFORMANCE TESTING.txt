PERFORMANCE TESTING — HEART DISEASE ANALYSIS PROJECT

1. Introduction
Performance Testing ensures that the Machine Learning model, Flask application, and dashboard perform efficiently under different inputs and conditions.  
This phase validates:
- Accuracy of predictions
- Response time
- Model stability
- System reliability

---------------------------------------------------------
2. Machine Learning Model Performance
---------------------------------------------------------

A) Accuracy Score
The model was trained on 4500 records with 18 features.
Final Model Used: Logistic Regression / Random Forest  
Accuracy Achieved: 85–90% (approx based on dataset behavior)

B) Evaluation Metrics
- Precision: Measures how many predicted positives are correct.
- Recall: Measures how many actual positives were captured.
- F1-Score: Balance between Precision & Recall.
- Confusion Matrix: Shows TP, FP, TN, FN.

Sample Interpretation:
- High True Negatives → model is good at detecting “No Heart Disease”
- True Positives → identifies at-risk individuals correctly

C) Cross Validation Results
- 70–80% consistent performance across 5 folds
- Indicates model stability and reliability
